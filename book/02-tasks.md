# 2. 벤치마크 과제 3종 — 선정 이유와 기대효과

비교 **대상**은 도구 2개(codex · openhands)이고, 그 둘에게 **똑같이 시키는 과제가 3종**이다.
왜 하필 이 3개인가? 그리고 각 과제에서 무엇을 보려고 하는가(기대효과)?

## 한눈에

| 레벨 | 과제 | 복잡도 차원 | 채점 방식 | 이 레벨이 가리는(분별하는) 것 |
|------|------|-------------|-----------|------------------------------|
| **L1** | `fib(n)` 단일 파일 | single-file | judge가 `fib` 를 **import** 해 값 검증 | 기본기 — 함수 하나를 정확히 쓰는가 |
| **L2** | `wordstat` 멀티파일 CLI | multi-file CLI | judge가 **subprocess** 로 실행, stdout/종료코드 검증 | 협력 — 여러 파일로 나누고 표준출력 계약을 지키는가 |
| **L3** | KV 스토어 멀티모듈 서비스 | multi-module service | judge가 **별도 프로세스**로 CLI 호출, 영속·종료코드 계약 검증 | 완성력 — 패키지 + 상태 영속까지 끝까지 만드는가 |

## 왜 "3단계 사다리"인가

핵심 아이디어: **난이도를 한 칸씩 올려가며 도구의 한계가 어디서 드러나는지**를 본다.

- **한 과제만 쓰면** 우연(쉬워서 둘 다 통과 / 어려워서 둘 다 실패)에 묻혀 차이가 안 보인다.
- **3단계로 벌리면** "어디까지는 둘 다 되고, 어디서부터 갈라지는가"라는 **분기점**이 보인다.
- 각 단계는 **앞 단계가 요구하지 않던 새 능력**을 추가로 요구하도록 설계했다:
  - L1 → L2: 파일 1개 → **여러 파일 협력 + CLI 인터페이스 계약**
  - L2 → L3: 명령 실행 → **패키지 구조 + 프로세스 간 상태 영속**

> 그래서 결과를 보면 "능력의 천장"이 어디인지 읽힌다. 둘 다 L1·L2는 통과하는데 L3에서 갈린다면,
> 그 도구의 한계는 **대형·다구성 과제를 끝까지 완성하는 지점**에 있다는 뜻이다.

## 왜 이 구체적 과제(fib / wordstat / KV)인가

- **fib(피보나치)** — 군더더기 없는 단일 함수. "도구가 기본적인 코드를 정확히 쓰는가"의 바닥선.
- **wordstat(단어 통계 CLI)** — 입력 파싱·집계·정렬·표준출력이라는 *현실적인 CLI* 의 최소형. 여러
  파일로 나누는 구조와 출력 형식 계약을 동시에 본다.
- **KV 스토어(키-값 서비스)** — 패키지(모듈 여러 개) + **디스크 영속 + 프로세스 재시작 후에도
  데이터 유지** 라는, 실제 서비스에 가까운 요구. "장난감"을 넘는 완성도를 본다.

또한 세 과제 모두 **Python 표준 라이브러리만** 쓴다 — 외부 설치 없이 **어디서나 재현**되게 하기 위해서다.

## 공정성 장치 (모든 레벨 공통)

- **프롬프트는 한 곳에만**(`benchmark/tasks/<level>/PROMPT.md`), 두 도구에 **글자 그대로** 동일하게 준다.
- **채점기(judge)는 도구에게 숨긴다** — 프롬프트는 테스트 내용을 모른다. 그래서 한 채점기로 어느
  도구의 산출물이든 공정하게 판정하고, 도구가 테스트에 과적합할 수 없다.
- L2·L3는 **블랙박스**(subprocess)로 채점한다 — 내부 구현이 어떻든 *겉으로 약속한 동작*만 본다.

## 기대효과 (이 3종으로 무엇을 알게 되나)

| 무엇을 알고 싶나 | 어느 레벨이 답하나 |
|------------------|--------------------|
| 두 도구가 **기본 코드**를 정확히 쓰는가 | L1 (둘 다 통과가 기대치) |
| **구조화된 CLI**를 만들고 출력 계약을 지키는가 | L2 |
| **복잡한 멀티모듈 서비스를 끝까지 완성**하는가 (도구의 천장) | L3 (여기서 갈릴 가능성↑) |
| 같은 능력이라도 **얼마나 빠른가** | 모든 레벨의 Time |
| 작업을 **어떤 식으로** 진행하는가(거칠기) | 모든 레벨의 Steps(도구 내부 추세) |

> 요약하면: **L1=바닥선, L2=현실적 중간, L3=천장 시험.** 두 도구가 이 사다리의 어디서 갈라지는지가
> 이 벤치마크의 핵심 관전 포인트다.
